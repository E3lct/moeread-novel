### 部署 + 章节识别改进 + 摸鱼模式修复（6月28日凌晨续）
- **部署到 Tomcat**：编译 StealthServlet.java（修复 `var` 关键字 → `List<Chapter>` 显式类型，JDK1.8 不支持 var）；同步所有新文件（stealth.jsp/css、reader.jsp/css、book.jsp/_book_card.jsp、bookshelf.css、ImportService.class、web.xml）到 Tomcat webapps/moeread；web.xml 添加 /stealth 到 LoginFilter 拦截；备份原 moeread.war 为 .bak 避免重启时重新解包覆盖
- **章节识别错乱问题解决**：用户报告导入小说章节错乱。根因：旧 ImportService 正则只匹配"第X章/回/节"，正文里偶然出现"第三章""第五章"等引用会被误识别为新章节。借鉴 Legado 阅读 APP（gedoor/legado，已因版权下架源码删除）算法思路重写 parseChapters：
  - 4 条正则规则：第X章/回/节/卷/集/部/篇 + 卷X + Chapter N + 楔子/序章/前言/后记/尾声/番外/外传/终章
  - 行长度过滤（≤30 字符），避免误识别正文长句
  - 超长章节（>3万字）按段落断点自动拆分
  - 写了 ChapterParserTest 测试 6 个用例全部通过，关键测试：正文含"第三章""第五章"引用正确识别为 1 章而非 3 章
- **摸鱼模式 stealth.jsp 修复**：
  - 修复 JS `getElementById('stRoot')` 引用错误（实际 id 是 stealthRoot），导致主题切换失败
  - 标签栏右侧添加"‹ 上一章 | 下一章 ›"导航按钮（伪装成 VS Code tab 操作）
  - 键盘 Alt+←/→ 翻章跳转实现（之前只 history.back / 空注释）
  - ESC 键返回书架（假装关掉编辑器）
  - 主题选择 localStorage 持久化
- **阅读器 reader.jsp**：顶部栏右侧添加"摸鱼"入口按钮，跳转 /stealth?bookId=xx&chapter=N
- **编译技巧**：Git Bash 下 javac -cp 用分号会被 bash 当命令分隔符，改用 PowerShell `& javac -cp $cp` 或 cmd //c "set CP=... && javac -cp \"%CP%\""；用通配符 `*.java` 编译时 StealthServlet.class 可能不生成（原因不明），单独编译即可
- **Legado 调研结论**：gedoor/legado 仓库 main 分支只剩README版权公告，源码已全部删除，无法直接 fork；但 Legado 章节识别本质就是"正则规则集 + 行长度过滤 + 超长拆分"，自行实现等价算法更实用
- **未解决问题**：测试6 中 `【第二章】风云再起` 这种带方括号装饰的标题未被识别（正则未覆盖方括号），可后续优化但不影响主流格式
